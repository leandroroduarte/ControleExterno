💾 BANCO DE DADOS
═══════════════════════════════════════════════════════════════════════════════

CREDENCIAIS PADRÃO:
   Host:     127.0.0.1
   Porta:    3306
   Database: ControleExterno
   User:     root
   Password: (vazio)

ESTRUTURA DA TABELA:
   Nome da Tabela: Users
   
   Campos:
   ├─ id (INT) .................... Identificador único, chave primária
   ├─ nome (VARCHAR 255) .......... Nome completo do usuário
   ├─ email (VARCHAR 255) ......... Email único e não nulo
   ├─ senha (VARCHAR 255) ......... Senha criptografada com bcrypt
   ├─ tipo (ENUM) ................. 'user' ou 'admin'
   └─ created_at (TIMESTAMP) ...... Data de criação automática

PARA CRIAR A TABELA:
   1. Execute o arquivo: database_setup.sql
   2. Ou copie e execute os comandos SQL manualmente
   3. Verifique se a tabela foi criada

🔒 MEDIDAS DE SEGURANÇA IMPLEMENTADAS
═══════════════════════════════════════════════════════════════════════════════

1. CRIPTOGRAFIA
   ├─ password_hash() com algoritmo bcrypt
   ├─ password_verify() para validação
   └─ Nenhuma senha é armazenada em texto plano

2. PROTEÇÃO SQL
   ├─ Prepared statements em todas as queries
   ├─ Placeholders (?) para parâmetros
   └─ Proteção contra SQL Injection

3. PROTEÇÃO XSS
   ├─ htmlspecialchars() em todos os outputs
   ├─ Escape de dados antes de exibir
   └─ Proteção contra scripts maliciosos

4. GERENCIAMENTO DE SESSÃO
   ├─ $_SESSION para armazenar dados de usuário
   ├─ session_destroy() no logout
   ├─ Sessão única por navegador
   └─ Expiração ao fechar o navegador

5. VALIDAÇÃO
   ├─ FILTER_VALIDATE_EMAIL para emails
   ├─ Requisito de senha mínima (6 caracteres)
   ├─ Verificação de email único
   ├─ Confirmação de senha obrigatória
   └─ Validação de campos vazios

6. PROTEÇÃO CSRF
   ├─ Uso de sessões PHP
   ├─ POST para operações sensíveis
   └─ Validação de origem de requisições

═══════════════════════════════════════════════════════════════════════════════
🆘 TROUBLESHOOTING E SOLUÇÕES
═══════════════════════════════════════════════════════════════════════════════

PROBLEMA: "Erro no banco de dados"
SOLUÇÃO:
   1. Verifique se MySQL/MariaDB está rodando
   2. Confirme se criou o banco 'ControleExterno'
   3. Verifique as credenciais no código
   4. Execute o arquivo database_setup.sql

PROBLEMA: "Email ou senha inválidos"
SOLUÇÃO:
   1. Verifique a digitação exata
   2. Certifique-se de ter cadastrado a conta
   3. Lembre que é case-sensitive
   4. Limpe o cache do navegador

PROBLEMA: "Email já cadastrado"
SOLUÇÃO:
   1. Use outro email para nova conta
   2. Ou faça login com este email existente
   3. Para reset de senha, atualize direto no banco

PROBLEMA: "Página em branco"
SOLUÇÃO:
   1. Abra o console do navegador (F12)
   2. Procure por erros JavaScript
   3. Verifique os logs do servidor PHP
   4. Confirme permissões de arquivo

PROBLEMA: "Não consigo fazer logout"
SOLUÇÃO:
   1. Clique em "Sair" na navbar
   2. Ou acesse: logout.php
   3. Ou feche o navegador completamente

💡 DICAS E BOAS PRÁTICAS
═══════════════════════════════════════════════════════════════════════════════

1. EMAIL ÚNICO
   → Cada email só pode ser cadastrado uma vez
   → Se esquecer a senha, atualize no banco de dados

2. SENHAS
   → Mínimo 6 caracteres obrigatório
   → Use senhas fortes com maiúsculas e números
   → Não é possível recuperar senha, apenas resetar

3. TIPOS DE USUÁRIO
   → Definido no cadastro e pode ser alterado no banco
   → UPDATE Users SET tipo='admin' WHERE id=1;

4. SESSÕES
   → Expira ao fechar o navegador
   → Uma sessão por aba/navegador
   → Logout destroi completamente

5. BANCO DE DADOS
   → Sempre faça backups regulares
   → Nunca use root sem senha em produção
   → Configure usuarios com permissões limitadas

6. PRODUÇÃO
   → Mude as credenciais do banco
   → Configure HTTPS
   → Adicione rate limiting
   → Configure logs de segurança
   → Implemente backup automático
